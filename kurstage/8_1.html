Auftrag 8.1: Dein Code geht online (GitHub Pages)

Dein Ziel

Dein Code ist im Moment nur auf deinem Laptop. Das ändern wir jetzt. Du erstellst dein erstes Portfolio, indem du deine Projekte aus Woche 6/7 veröffentlichst. Das Ziel ist es, zwei Links zu haben, die du mit der Klasse teilen kannst: eine echte, klickbare Webseite für deine Web-App und eine professionelle Projektseite für dein Python-Spiel.

1 GitHub-Konto erstellen

Falls du noch kein Konto hast, lass uns das zuerst erledigen. Die KI führt dich durch die Anmeldung.

Hallo! Du bist ein/e GitHub-Experte/in. Ich bin Anfänger/in und habe noch kein GitHub-Konto. Bitte führe mich Schritt für Schritt durch den Prozess der Kontoerstellung auf github.com.

2 Web-App (HTML/JS) veröffentlichen

Jetzt bringen wir deine interaktive Webseite (z.B. To-Do-Liste) live ins Internet. Am Ende dieses Schritts hast du einen funktionierenden Link.

Ich habe jetzt ein GitHub-Konto. Leite mich bitte Schritt für Schritt an, um meine HTML/JS-Webseite live zu schalten.

1. Zeige mir, wie ich ein NEUES, öffentliches Repository für meine Webseite erstelle (z.B. mit dem Namen `meine-web-app`).
2. Zeige mir, wie ich meine `index.html`-Datei (und eventuelle CSS/JS-Dateien) per "Upload files" im Browser in dieses Repository hochlade.
3. Führe mich durch die "Settings" -> "Pages", um GitHub Pages für die `main`-Branch zu aktivieren.
4. Erkläre mir, dass es ein paar Minuten dauern kann, und wo ich den finalen Link finde (z.B. `benutzername.github.io/meine-web-app`).

3 Python-Projekt präsentieren

Dein Python-Spiel kann nicht direkt im Browser laufen. Stattdessen erstellen wir eine professionelle Projektseite, die deinen Code und eine Beschreibung anzeigt.

Ich habe meine Web-App veröffentlicht. Jetzt möchte ich mein Python-Projekt (ein Text-Adventure) auf einer Projektseite präsentieren.

1. Zeige mir, wie ich ein ZWEITES, öffentliches Repository für mein Python-Projekt erstelle (z.B. `mein-python-spiel`).
2. Hilf mir, eine professionelle `README.md`-Datei zu schreiben. Frage mich nach dem Namen meines Spiels und was es tut, und erstelle dann den Markdown-Code für eine gute Beschreibung.
3. Zeige mir, wie ich meine Python-Datei (z.B. `app.py`) UND die `README.md`-Datei in dieses zweite Repo hochlade.
4. Zeige mir, wie ich auch für dieses Repo GitHub Pages aktiviere, damit die `README.md` schön angezeigt wird.

✅ Ziel erreicht!

Du bist fertig, wenn du zwei Links hast und sie der Klasse zeigen kannst:

  • Einen Link zu deiner live-Web-App.
  • Einen Link zur Projektseite deines Python-Spiels.
